## 一、基础概念一分钟科普

在动手之前，你需要了解几个最基本的概念。不需要记住公式，只需要理解它们在说「什么」。

### 1.1 什么是辐射制冷？

想象一下夏天的夜晚，你躺在院子里仰望天空。你感觉凉快，不仅因为没有太阳了，还因为**你的身体在持续向天空散发红外线热量**。太空很冷（接近零下270°C），所以红外线会一直往外跑，带走热量。

辐射制冷就是利用这个原理：设计一种材料或涂层，让它**尽可能多地向太空发射红外线，同时尽可能少地吸收太阳光和大气辐射**，这样材料就能持续降温。

### 1.2 衡量制冷效果的核心指标 — P_net

「这个东西能让我多凉快？」用一个数字回答就是 **净制冷功率 P_net**（单位：瓦/平方米，W/m²）。

你可以把它理解成：**每平方米每秒能从你身上带走多少瓦的热量**。

- P_net > 0：净制冷（降温）
- P_net = 0：既不制冷也不制热
- P_net < 0：净制热（升温）

### 1.3 影响 P_net 的四个因素

```
P_net = 辐射出去的热量 - 吸收进来的热量
```

**辐射出去** — 材料「发射红外线」的能力（发射率 ε，越高越好）

**吸收进来的热量**有三个来源：

| 来源        | 说明                   | 理想情况                         |
| ----------- | ---------------------- | -------------------------------- |
| 大气辐射    | 大气层向你发射的红外线 | 大气透光好（透明窗口），红外线少 |
| 太阳光      | 白天的太阳辐射         | 反射掉太阳光（高太阳反射率）     |
| 热传导/对流 | 风把热量吹进来         | 没有风，或者隔热做得好           |

### 1.4 辐射制冷的两大天敌

| 天敌              | 效果                               | 对策                             |
| ----------------- | ---------------------------------- | -------------------------------- |
| **太阳**    | 白天太阳辐射很强，会加热材料       | 高太阳反射率（白色/镜面涂层）    |
| **水汽/云** | 大气中的水汽会「反弹」红外辐射回来 | 利用大气透明窗口波段（8-13微米） |

### 1.5 白天 vs 夜晚：完全不同的挑战

| 时段           | 主要矛盾               | 关键参数                                      |
| -------------- | ---------------------- | --------------------------------------------- |
| **夜晚** | 大气辐射是主要热量来源 | 高发射率、低水汽地区效果好                    |
| **白天** | 太阳辐射是主要热量来源 | 高太阳反射率 + 高发射率才能同时「散热又隔热」 |

| **太阳加权反射率** R_sol | 反射率在 AM1.5 标准太阳光谱下的加权平均值                             | 系统自动计算       |

| **大气发射率** ε_atm    | 大气向地面辐射热量的等效发射率（与天气、湿度有关）                    | 系统根据预设计算   |

| **平衡温度** T_eq        | 当 P_net = 0 时，材料的温度                                           | 风速云图求解       |
| **大气窗口**             | 8-13 μm 波长范围，大气在此处最透明，红外热量可以「穿过」大气到达太空 | 物理常识           |

## 三、新建任务 — 制冷与制热计算

### 3.1 这是什么？

这是系统的**核心功能**：上传你的材料光谱数据，系统帮你计算这种材料的制冷（或制热）效果。

### 3.2 适用场景

- 你研发了一种新涂层，想知道它制冷效果如何
- 你在两种现有材料之间做选型对比
- 你想发论文，需要给出材料在特定条件下的制冷功率数据

### 3.3 使用步骤

**第一步：准备材料数据**

你需要准备 3 个文件（具体格式见第五章）：

| 文件           | 作用                                            | 例子                              |
| -------------- | ----------------------------------------------- | --------------------------------- |
| 反射率文件     | 告诉系统材料对太阳光（可见光+近红外）的反射能力 | 白漆反射90%的阳光                 |
| 发射率文件     | 告诉系统材料发射红外线的能力                    | 聚乙烯在8-13μm发射率高达0.95     |
| 大气透过率文件 | 告诉系统大气在红外波段透过去了多少              | 不同地区/季节的大气窗口透光率不同 |

**第二步：上传文件**

访问 `/uploads`，上传上述三个文件。上传后它们会自动变成「激活」状态，表示当前正在使用。

**第三步：创建任务**

访问 `/jobs/new`，选择任务类型：

| 任务类型           | 什么时候用                             |
| ------------------ | -------------------------------------- |
| **辐射制冷** | 你想让东西凉下来（降温）               |
| **辐射制热** | 你想让东西热起来（保温、太阳能热水器） |

**第四步：设置参数**

| 参数          | 说明                                  | 默认值           |
| ------------- | ------------------------------------- | ---------------- |
| 精度等级      | 计算精度。越高越精确，但越慢          | 中等             |
| 大气预设      | 模拟哪个地区/季节的大气条件           | 默认（中等湿度） |
| 启用自然对流  | 是否考虑材料表面和空气之间的热交换    | 启用             |
| 相变温度/功率 | 如果材料有相变特性（如PCM），填在这里 | 不填             |
| 启用蒸发冷却  | 如果材料表面有水膜，计算潜热效果      | 关闭             |
| 相对湿度      | 环境湿度，影响蒸发冷却                | 不填             |

**第五步：提交并等待结果**

点击「提交任务」，系统开始计算。任务会自动跳转到详情页，你可以看到：

- 任务状态（排队中 → 运行中 → 完成）
- 计算完成后的曲线图和数据表
- 下载 CSV/Excel 原始数据

### 3.4 结果怎么看？

**制冷任务** 会输出这张图：

- **横轴**：材料温度（假设它达到了多少度）
- **纵轴**：净制冷功率 P_net（W/m²）
- **多条线**：代表不同对流条件（无风、微风、强风）

**这张图告诉你的事**：

- 看某条线上 P_net > 0 的区间，就是这种材料「能制冷」的条件
- P_net 越大，制冷效果越好
- 温度升高，P_net 一般会下降（温差越大，辐射散热越难）
- 风越大，曲线越往下掉（风把热量吹进来）

### 4.4 结果解读四步法

拿到计算结果后，按以下顺序阅读：

**第一步：看功率-温度曲线（图表 1_1）**

这是最核心的结果。横轴是材料温度，纵轴是净制冷功率 P_net。

- **零功率线（Y=0）** 是制冷/制热的分水岭
- 曲线在零功率线以上 → 这种材料在此温度下能制冷
- **峰值制冷功率** 是材料能达到的最大制冷能力，数值越大越好

**典型判断标准**：

| 峰值制冷功率 | 效果评价 | 适合做什么                 |
| ------------ | -------- | -------------------------- |
| > 100 W/m²  | 优秀     | 显著降低建筑冷负荷         |
| 50-100 W/m² | 良好     | 被动制冷的有效辅助         |
| 10-50 W/m²  | 一般     | 仅夜间有效或需辅助手段     |
| < 10 W/m²   | 较差     | 材料性能不足或参数设置有误 |

**第二步：看功率分量图（图表 1_2）**

这张图把制冷功率分解为各个分量，帮助你诊断「为什么效果不好」：

- 如果 **Q_solar（太阳吸收）曲线** 在高温区上升最快 → 太阳反射率不够高，优先改进 R(λ) 数据
- 如果 **Q_conv（对流换热）曲线** 下降较快 → 对流损失严重，换低风速场景或加隔热层
- 如果 **纯辐射项曲线** 本身较低 → 红外发射率不足，检查 ε(λ) 是否覆盖 8-13 μm 窗口

**第三步：看风速-湿度影响图（图表 1_4）**

- **低风速区（< 1 m/s）** 代表无风条件下的制冷上限
- 等高线在风速方向密集 → 风速是主要限制因素
- 相对湿度 20%→80% 时，蒸发潜热先增后减（湿度过高反而无效）

**第四步：看小时变化统计（图表 2_1）**

如果运行了 ERA5 原位模拟，这张图展示一天中各时段的制冷功率变化：

- **夜间（22:00-06:00）**：无太阳热负荷，制冷功率稳定
- **白天（08:00-16:00）**：太阳辐照度变化剧烈，中午功率最低甚至转为制热
- **日落前后（17:00-19:00）**：太阳辐照度快速下降，制冷功率迅速回升

| 问题                       | 可能原因                        | 解决思路                                                                  |
| -------------------------- | ------------------------------- | ------------------------------------------------------------------------- |
| P_net 全是负值（无法制冷） | 太阳吸收率太高或红外发射率太低  | 检查反射率文件是否正确；确认波长范围是否 300-2500 nm                      |
| 计算结果与文献数据差异大   | 大气预设不匹配                  | 尝试不同大气预设；检查 T_a 设置                                           |
| 夜间制冷功率偏低           | 大气发射率偏高或风速过高        | 用风速云图评估；确认无风基准功率                                          |
| 峰值温度位置不对           | 热连接参数（hc_values）设置不当 | 在 config.ini 中调整等效导热系数                                          |
| 透射率材料效果差           | 太阳吸收被低估                  | 同时上传 reflectance + transmittance 文件                                 |
| 文件显示「波长超出范围」   | 数据范围与系统默认不匹配        | 系统默认太阳波段 300-2500 nm，红外 8-13 μm；可上传自定义波长网格文件覆盖 |

## 六、工具箱

工具箱是系统最有价值的部分，每个工具都针对一个特定的研究问题而设计。

> **如何进入**：登录后访问 `/tools`

### 6.1 风速云图 — 风大时制冷效果会变差多少？

#### 6.1.1 这个问题是什么？

辐射制冷材料在户外使用时，风是影响效果的重要因素。风会把热量「吹进」材料表面，削弱制冷效果。但风速在不同地区差异很大——海边经常有风，内陆风就小一些。

**这个工具帮你回答**：「如果我这种材料在有风的环境下使用，制冷效果会打多少折扣？」

#### 6.1.2 能干什么？

输入你的材料数据，设定风速范围和大气发射率范围，系统会计算出一个**二维云图**：

- **横轴**：风速（0到5米/秒，约等于0级到3级风）
- **纵轴**：大气发射率（代表不同湿度/云量条件）
- **颜色/数值**：达到热平衡时，材料比周围空气高（或低）多少度

#### 6.1.3 怎么看懂这张图？

**如果 ΔT > 0**（材料比空气热）：说明材料温度高于环境，靠辐射散不出去热量
**如果 ΔT < 0**（材料比空气冷）：说明材料在制冷，数值越负，制冷越强

💡 **实际意义**：ΔT 约等于材料能达到的降温幅度（与环境温度的差值）。

#### 6.1.4 输入参数

| 参数       | 说明                     | 建议                         |
| ---------- | ------------------------ | ---------------------------- |
| 风速范围   | 0-5 m/s 通常覆盖日常场景 | 可以按需扩大                 |
| 发射率范围 | 0-1，默认 0-1            | 通常不需要改                 |
| 太阳辐照度 | 可选，留空使用默认值     | 晴天的典型值约800-1000 W/m² |

#### 6.1.5 典型应用案例

**场景**：你设计了一款辐射制冷涂料，想评估它在兰州（多风城市）和成都（少风城市）的表现差异。

**操作**：

1. 上传涂料的光谱数据
2. 设定风速范围：0-8 m/s
3. 发射率保持默认
4. 运行计算

**解读**：图中可以看出，在兰州（年平均风速约3 m/s），ΔT 可能只有 -2°C；而在成都（年平均风速约1 m/s），ΔT 可能达到 -5°C。这说明风速对这款涂料效果影响很大。

#### 6.1.6 参数设置指南

| 参数           | 推荐范围             | 怎么选                                                    |
| -------------- | -------------------- | --------------------------------------------------------- |
| 风速范围       | 0-10 m/s             | 大多数建筑场景 < 5 m/s；高处安装（楼顶、风口）可到 10 m/s |
| 大气发射率范围 | 0.3-1.0              | 晴朗干燥大气：0.65-0.80；高湿度大气：0.40-0.60            |
| 采样点数       | 50-100               | 100 足够，无需超过 200                                    |
| 太阳辐照度     | null 或 0-1000 W/m² | 评估夜间性能设为 null（自动置零）；白天场景设为 800-900   |

💡 **快速技巧**：将风速固定为 0（即无对流损失），作为材料性能上限的参考基准。

#### 6.1.7 如何理解等高线形状？

| 等高线形状                     | 含义                                  | 对应场景             |
| ------------------------------ | ------------------------------------- | -------------------- |
| **倒碗形**（向右下凹陷） | 正常形状，风速↑+发射率↓都会削弱制冷 | 典型场景             |
| **几乎垂直**             | 对流对结果影响小，辐射项占主导        | 高性能材料           |
| **几乎水平**             | 发射率已饱和，其他因素在限制          | 可能存在太阳吸收问题 |

- **前端**：`/tools/wind-cloud`
- **后端**：`POST /api/tools/wind-cloud`
- **核心函数**：`webapi/services/tool_service.py::compute_wind_cloud()`

### 6.2 太阳能效率 — 这种材料在太阳底下是制冷还是制热？

#### 6.2.1 这个问题是什么？

白天和夜晚的挑战完全不同。白天太阳很强（可达 1000 W/m²），大部分材料会被太阳晒热。问题在于：**到底是太阳加热的效果强，还是辐射制冷的效果强？**

**这个工具帮你回答**：「在给定的太阳强度和环境温度下，我的材料是净制冷还是净制热？边界在哪里？」

#### 6.2.2 能干什么？

这个工具在**温度-太阳辐照度**构成的二维平面上，绘制材料的净制热功率等值线图。

- **横轴**：环境温度（从 -40°C 到 60°C）
- **纵轴**：太阳辐照度（从 0 到 1200 W/m²）
- **颜色**：净制热功率（正值=制热，负值=制冷）

#### 6.2.3 怎么看懂这张图？

图中有几个关键信息：

1. **零线（交界线）**：P_heat = 0 的那条等值线，是「制冷/制热」的分界线
2. **斜率**：交界线的斜率约等于材料的太阳吸收率（α_s）。斜率越大，说明越容易被太阳晒热
3. **等值线间距**：等值线越密，说明该区域性能对参数越敏感

#### 6.2.4 典型应用案例

**场景**：你有一种新材料，想知道它在夏季晴天（太阳辐照度约 800 W/m²）和冬季晴天（约 500 W/m²）分别是什么效果。

**操作**：

1. 上传材料光谱
2. 温度范围设定 -20°C 到 60°C
3. 太阳辐照范围设定 0 到 1200 W/m²
4. 运行

**解读**：

- 如果点 (30°C, 800 W/m²) 落在负值区域 → 夏季晴天能制冷
- 如果点 (5°C, 500 W/m²) 落在正值区域 → 冬季晴天是制热效果

#### 6.2.5 与风速云图的区别

| 工具       | 核心问题                   | 适合回答               |
| ---------- | -------------------------- | ---------------------- |
| 风速云图   | 「有风时效果打多少折扣？」 | 评估户外场景           |
| 太阳能效率 | 「白天是制冷还是制热？」   | 评估日夜差异、季节差异 |

#### 6.2.6 参数设置指南

| 参数           | 推荐范围        | 怎么选                                                        |
| -------------- | --------------- | ------------------------------------------------------------- |
| 环境温度范围   | -30°C 到 60°C | 涵盖建筑热舒适范围；极寒地区可扩展到 -40°C                   |
| 太阳辐照度范围 | 0-1200 W/m²    | 0=夜间基准；400-600=阴天；800-1000=晴天正午；1100+=高海拔晴天 |

**各区域物理含义**：

- 左上角（S≈0，低温）：纯辐射散热条件，夜间或阴天制热效率
- 右下角（S≈1000，高温）：夏季正午极端场景
- P_heat > 0：材料积累热量，有利于冬季供暖
- P_heat < 0：即使有太阳仍在散热，不利于供暖但可能是设计目标

#### 6.2.7 代码入口

- **前端**：`/tools/solar-efficiency`
- **后端**：`POST /api/tools/solar-efficiency`
- **核心函数**：`webapi/services/tool_service.py::compute_solar_efficiency()`

### 6.3 发射率-太阳云图 — 材料能制冷的条件是什么？

#### 6.3.1 这个问题是什么？

对于一种特定材料，在什么条件下它能实现制冷？比如：**如果大气比较潮湿（红外辐射强），或者太阳特别晒，还能不能制冷？**

**这个工具帮你回答**：「我的材料在哪些「大气条件 + 太阳强度」组合下，能做到净制冷？」

#### 6.3.2 能干什么？

以**大气发射率**和**太阳辐照度**为双参数，绘制一张相图，图中清晰标注了「能制冷 / 不能制冷」的分界线。

- **横轴**：大气有效发射率（代表大气湿度/云量）
- **纵轴**：太阳辐照度
- **颜色/数值**：净制冷功率
- **零等值线**：制冷/制热的临界边界

#### 6.3.3 怎么看懂这张图？

**零等值线（分界线）的含义**：

- 分界线以上：太阳太晒，即使大气条件再好也无法制冷
- 分界线以下：材料可以净制冷
- 分界线越靠上，说明材料「抗晒」能力越强（高反射率）

#### 6.3.4 典型应用案例

**场景**：你想知道你的材料在以下三种典型天气下的表现：

- 晴朗干燥（大气发射率低，太阳辐照度高）
- 多云潮湿（大气发射率高，太阳辐照度中）
- 阴天（大气发射率高，太阳辐照度低）

**解读**：

- 晴朗干燥：A点 → 高太阳 + 低大气发射率 → **大多数材料无法制冷**
- 多云潮湿：B点 → 中等太阳 + 高大气发射率 → **取决于材料反射率**
- 阴天：C点 → 低太阳 + 高大气发射率 → **大多数材料可以制冷**

这正好说明了为什么辐射制冷在阴天反而比晴天容易成功。

#### 6.3.5 零等值线为什么是弯曲的？

等高线图中的 P_cool = 0（零功率线）将参数空间划分为制冷区和制热区。曲线的弯曲形状反映了太阳吸收与辐射制冷之间的非线性竞争。

- **高大气发射率（高湿度/多云）** 时，窗口制冷效应减弱，零功率线向左移动
- **低大气发射率（干燥晴朗）** 时，零功率线向右移动，允许更高太阳辐照度

💡 **快速判断**：找到零功率线对应的太阳辐照度，那就是该地区白天制冷所需的"最低太阳反射率门槛"。高于此值才能白天制冷。

- **前端**：`/tools/emissivity-solar`
- **后端**：`POST /api/tools/emissivity-solar`
- **核心函数**：`webapi/services/tool_service.py::compute_emissivity_solar_cloud()`

### 6.4 功率分量 — 制冷效果不好，是哪个环节拖了后腿？

#### 6.4.1 这个问题是什么？

系统计算出的净制冷功率 P_net 是很多因素叠加后的结果。有时候 P_net 很低，但你不知道为什么——是辐射散热不够？是太阳吸热太多？还是风太大？

**这个工具帮你回答**：「我的材料，**每一项**热量收支分别是多少？」

#### 6.4.2 能干什么？

把净制冷功率 P_net **拆解成 8 个独立分量**，画成一张多线图：

| 分量                      | 物理含义                 | 正负含义                           |
| ------------------------- | ------------------------ | ---------------------------------- |
| p_r（向外辐射）           | 材料发射红外线向外散热   | 恒正，散热                         |
| p_a（大气辐射）           | 大气向材料发射红外线加热 | 恒正（大气在向你「倒热」）         |
| Q_conv（对流散热）        | 材料表面与空气的热交换   | 温差越大/风速越大，值越大          |
| Q_solar（太阳吸收）       | 太阳光被材料吸收转化为热 | 太阳辐照度越高，值越大             |
| P_phase（相变）           | 相变材料释放/吸收潜热    | 降温时正（放热），升温时负（吸热） |
| Q_latent（蒸发潜热）      | 材料表面水膜蒸发带走热量 | 湿度低/有水膜时正                  |
| **p_net（净功率）** | 以上所有分量的代数和     | 正=制冷，负=制热                   |

**核心定义（多线图中的辅助参数）：**

- **P_solar（太阳加热功率）**：太阳辐射功率，单位 W/m²。
- **Q_latent（蒸发潜热）**：材料表面水膜蒸发所带走的热量，与空气湿度差和蒸发系数有关，单位 W/m²。开启「蒸发冷却」功能时显示。
- **h_nat（自然对流传热因子）**：基于水平面自然对流关联式（层流 Nu=0.54·Ra^0.25，湍流 Nu=0.15·Ra^(1/3)）计算的自然对流换热系数，随温差（T_a - T_film）变化，单位 W·m⁻²·K⁻¹。默认按 10cm×10cm 方形样品（L_char=0.025m）计算。用于表征无风条件下的对流散热能力。
- **h_total（总传热因子）**：自然对流因子 h_nat 与等效导热/换热系数 h_cond 的综合，即 h_total = h_nat + h_cond，单位 W·m⁻²·K⁻¹。总对流散热 Q_conv = h_total · (T_a - T_film)。

#### 6.4.3 怎么看懂这张图？

**诊断方法**：

1. 看 p_r 曲线：高发射率材料的 p_r 曲线应该越高越好
2. 看 p_a 曲线：大气辐射越小越好（大气发射率低时）
3. 看 Q_solar：太阳吸收越小越好（太阳反射率高时）
4. 看 p_net 曲线：这就是最终的净效果

**诊断实例**：

- **现象**：p_net 很低，但 p_r 曲线本身很高
- **诊断**：说明你的材料辐射散热没问题，但被其他项（p_a / Q_solar / Q_conv）抵消了
- **对策**：如果 Q_solar 是主要抵消项 → 提高太阳反射率；如果 p_a 很大 → 选择干燥地区使用

#### 6.4.4 典型应用案例

**场景**：你的材料发射率很高（0.95），但实际制冷效果却一般。想知道原因。

**操作**：

1. 上传材料
2. 设定参数（精度、导热系数等）
3. 开启「功率分量」分析

**解读**：

- 可能发现 Q_solar = 50 W/m²（材料吸收了一半的太阳光）
- 如果把太阳反射率从 0.7 提高到 0.9，Q_solar 从 50 降到 30 W/m²
- p_net 增加了 20 W/m² → 这就是改进的方向

#### 6.4.5 额外功能

这个工具还支持：

- **相变材料分析**：如果你的材料有相变特性（如在某个温度下熔化/凝固），可以设置相变参数
- **蒸发冷却分析**：如果材料表面有水膜（如亲水涂层），可以开启潜热计算

#### 6.4.6 快速诊断流程

按以下顺序排查 P_net 不理想的原因：

1. **先确认太阳反射率 R_sol** — 白天制冷的第一要素，要求 > 0.90
2. **再检查红外发射率 ε** — 在 8-13 μm 窗口是否 > 0.80
3. **然后看风速设置** — 尝试设为 0，评估无对流条件下的基准功率
4. **最后检查大气预设** — 干燥地区用低 ε_atm 预设，潮湿地区用高 ε_atm 预设

**等效导热系数（h_cond）的选择**：

| h_cond        | 场景       | 含义                           |
| ------------- | ---------- | ------------------------------ |
| 0             | 完全隔热   | 材料自持温度，与环境无导热连接 |
| 5-10 W/m²·K | 典型室内   | 与墙体/屋顶有一定热连接        |
| > 50 W/m²·K | 良好热连接 | 材料温度紧跟环境温度，难以制冷 |

### 6.5 角度功率 — 应该朝哪个方向散热效果最好？

#### 6.5.1 这个问题是什么？

辐射制冷材料向四面八方都在散热。但不同方向的大气「质量」不一样——**朝天空正上方（天顶）散热效果最好**（那里是太空），而朝向地面的部分则会从地面吸收热量。

**这个工具帮你回答**：「材料的各个方向，散热效果分别是多少？设计罩子或百叶时，朝哪个方向开口最有利？」

#### 6.5.2 能干什么？

计算并可视化材料在**各个天顶角方向**上的辐射散热能力。

- **横轴**：天顶角（0° = 正对天空，90° = 正对地面）
- **纵轴**：单位立体角散热功率（W/m²/sr）

#### 6.5.3 怎么看懂这张图？

**关键信息**：

- **0°附近（天顶）**：数值最大，正值，表示向太空方向散热效果最好
- **90°附近（水平）**：数值接近0
- **如果曲线穿过零线**：说明某些角度是净吸热（从地面或建筑物吸热）

#### 6.5.4 典型应用案例

**场景**：你设计了一个辐射制冷屋顶，需要加装遮阳棚来遮挡太阳。但遮阳棚会挡住部分「天空视野」。你想知道：**遮挡多少角度以内还能保持较好的制冷效果？**

**操作**：

1. 输入材料发射率数据
2. 设定温差（通常为0，即材料温度=环境温度）
3. 运行后查看曲线

**解读**：

- 如果曲线在 0°-60° 区间都保持正值 → 说明只要遮阳棚不遮挡 60° 以内的天空，制冷效果就有保障
- 如果 30° 以下曲线就掉到零附近 → 说明这种材料需要「开阔的天空」才能发挥效果

#### 6.5.5 与其他工具的区别

| 工具               | 回答的问题                             |
| ------------------ | -------------------------------------- |
| 风速云图           | 风速影响有多大？                       |
| 太阳能效率         | 白天/太阳影响有多大？                  |
| 功率分量           | 各个热量来源/去向分别多少？            |
| **角度功率** | **各个方向散热效果有什么不同？** |

#### 6.5.6 极坐标图形状的含义

将角度-功率数据用极坐标表示，可以直观看到材料的"辐射方向性"：

| 形状                 | 含义       | 说明                                             |
| -------------------- | ---------- | ------------------------------------------------ |
| **圆形**       | 各向同性   | 材料在所有方向上均匀辐射（理想情况）             |
| **椭圆形**     | 角度依赖性 | 存在优先发射方向，与材料电磁特性或表面粗糙度有关 |
| **不对称图案** | 测量误差   | 可能存在非均匀性，建议检查材料样品               |

### 6.6 材料-环境温度云图 — 这种材料在哈尔滨和在海口表现一样吗？

#### 6.6.1 这个问题是什么？

中国幅员辽阔，哈尔滨冬季气温低至 -30°C，海口冬季气温在 15°C 以上。**同一种材料，在不同环境温度下，制冷效果完全不同**——因为辐射散热和对流散热的驱动力都和环境温度有关。

**这个工具帮你回答**：「我的材料在特定对流条件下，在不同环境温度下分别能发出多少制冷功率？」

#### 6.6.2 能干什么？

在**环境温度**（X轴）和**材料温度**（Y轴）构成的二维平面上，绘制净制冷功率云图。

- **横轴**：环境温度（-20°C 到 60°C，覆盖中国大部分地区）
- **纵轴**：材料温度（从你的材料光谱参数计算得到的范围）
- **颜色**：净制冷功率

#### 6.6.3 怎么看懂这张图？

**热力学直觉**：

- 材料温度远高于环境温度时 → 辐射散热驱动强，制冷效果好
- 材料温度接近环境温度时 → 散热驱动力弱，制冷效果差
- 材料温度低于环境温度 → 变成制热了

这张图帮你找到：**在某个环境温度下，材料温度设置在多少度时制冷效果最好**。

#### 6.6.4 典型应用案例

**场景**：你想知道你的辐射制冷涂层在哈尔滨（夏季白天约25°C）和海口（夏季白天约33°C）的表现差异。

**操作**：

1. 上传材料数据
2. 设定环境温度范围：10°C 到 40°C（覆盖两地夏季）
3. 设定对流换热系数：5 W/m²·K（模拟有轻微隔热层的情况）
4. 运行

**解读**：

- 哈尔滨（25°C）：材料在 20°C 时，p_net ≈ -40 W/m² → 较强制冷
- 海口（33°C）：材料在 28°C 时，p_net ≈ -25 W/m² → 制冷效果下降
- **结论**：海口温度高，制冷效果确实变差。但差别不大，说明这种材料两地都可用。

#### 6.6.5 参数设置指南

| 参数         | 推荐范围        | 怎么选                                                          |
| ------------ | --------------- | --------------------------------------------------------------- |
| 环境温度范围 | -20°C 到 45°C | 评估寒冷地区（青藏）设为 -30°C；评估炎热地区（中东）设为 45°C |
| 对流换热系数 | 0-30 W/m²·K   | h_cond=0: 隔热条件；h_cond=5-10: 背板连接（室内）；h_cond>15: 强风或机械通风；启用「自然对流」后额外叠加 h_nat（10cm样品约 3-12 W/m²·K） |

💡 **热连接的重要性**：h_c 越大，材料温度越接近环境温度。当 h_c 极高时（如强风），无论材料辐射性能多好，都无法与环境热侵袭抗衡，制冷功率趋近于零。

**用这张图筛选材料的方法**：

1. 确定目标气候的环境温度范围（如上海夏季 25-35°C）
2. 设定热连接条件（室内用 5-10 W/m²·K，室外屋顶用 10-20 W/m²·K）
3. 在云图中定位：在你的温度范围内，制冷功率是否均为正值
4. 若存在大面积负值区域，说明需要改进材料或辅助制冷手段

- **前端**：`/tools/material-env-temp-cloud`
- **后端**：`POST /api/tools/material-env-temp-cloud`
- **核心函数**：`webapi/services/material_env_temp_cloud.py::compute_material_env_temp_cloud()`

### 6.7 MODTRAN计算大气透过率 — 大气到底透过去了多少红外光？

#### 6.7.1 这个问题是什么？

大气不是完全透明的。在红外波段，大气中的水汽、二氧化碳、臭氧等会吸收和散射红外辐射。**搞清楚大气到底「放行了」多少红外光**，是准确模拟辐射制冷的基础。

**这个工具帮你回答**：「在特定大气条件下（热带/中纬度/高海拔等），8-13μm 大气窗口的透过率是多少？」

#### 6.7.2 能干什么？

调用专业的 **MODTRAN/PcMod5** 大气辐射传输模拟软件，计算并输出大气透过率光谱。

**输入**：

- 大气模型（热带/中纬度夏季/中纬度冬季/副极地夏季/副极地冬季/美国标准大气）
- 气溶胶模型（无/乡村/城市/海洋/对流层/沙漠）
- 观测几何（天顶角、方位角、地面高度等）
- 波数范围和分辨率

**输出**：

- 透过率光谱（CSV / Excel）
- 预览图表
- 可直接用于替代默认的大气透过率文件

#### 6.7.3 什么时候需要用这个？

**需要自定义大气参数时**，比如：

- 你的研究地点在高海拔（需要修改地面高度）
- 你的研究地点在沙漠（需要修改气溶胶模型）
- 你想模拟极端天气（极端湿度/云量）

**不需要自定义时**：

- 使用系统默认的大气预设就足够
- 大部分教学和初步分析场景

#### 6.7.4 典型应用案例

**场景**：你的研究在青藏高原（海拔约4500米），系统默认大气预设是海平面。你想模拟高海拔的干燥大气。

**操作**：

1. 选择大气模型：中纬度夏季
2. 气溶胶模型：选择「对流层气溶胶」
3. 地面高度：输入 4.5 km
4. 设定波数范围：400-5000 cm⁻¹（对应约 2.5-25 μm）
5. 运行

**输出**：下载生成的透过率文件，上传到材料管理系统，替换默认的大气参数。

#### 6.7.5 大气模型选择指南

| 模型         | 适用场景                        |
| ------------ | ------------------------------- |
| 热带         | 赤道附近（广州、海南、新加坡）  |
| 中纬度夏季   | 北纬30-45度夏季（华北、华东）   |
| 中纬度冬季   | 北纬30-45度冬季                 |
| 副极地夏季   | 北纬55-70度夏季（东北、新疆）   |
| 副极地冬季   | 北纬55-70度冬季（漠河、阿尔泰） |
| 美国标准大气 | 通用参考                        |

#### 6.7.6 气溶胶模型选择

| 气溶胶模型 | 适用场景                   |
| ---------- | -------------------------- |
| 无         | 高海拔清洁大气、实验室条件 |
| 乡村       | 远离工业的内陆地区         |
| 城市       | 大都市、工业区             |
| 海洋       | 沿海地区、高盐度空气       |
| 对流层     | 背景气溶胶、区域传输       |
| 沙漠       | 沙尘天气、干旱区沙暴       |

💡 **气溶胶对 8-13 μm 窗口的影响相对较小**（散射主要在可见光波段），但沙漠气溶胶中的大颗粒会引入额外吸收，建议在西北地区研究时选择「沙漠」模型。

#### 6.7.7 观测几何参数详解 — 它们如何影响大气模型？

前端界面中，有四个与几何视角相关的参数，它们共同决定了 MODTRAN 建立大气辐射传输路径的方式。这些参数最终被写入 MODTRAN 的第 4 行（`ground_alt`）和第 5 行（`observer_zenith`、`observer_azimuth`、`solar_zenith`、`solar_azimuth`），直接控制大气柱的路径长度和方向。

**参数流向总览**：

```
前端 UI  →  API (/api/tools/modtran/run)  →  LtnParams dataclass
  →  modtran_ltn.py (写入 .ltn 模板第4/5行)
  →  PcMod5.exe (MODTRAN 大气辐射传输计算)
  →  tape6/tape7 输出文件
```

这些参数在 `LtnParams` 中的对应关系：

```python
@dataclass
class LtnParams:
    observer_zenith_deg: float   # → 第5行第0字段：观测天顶角
    observer_azimuth_deg: float  # → 第5行第1字段：观测方位角
    solar_zenith_deg: float      # → 第5行第2字段：太阳天顶角
    solar_azimuth_deg: float     # → 第5行第3字段：太阳方位角
    ground_alt_km: float         # → 第4行第10字段：地面高度
```

##### 观测方位角 (Observer Azimuth)

**含义**：地表观测者"朝哪个方向"看天空时的罗盘方向（顺时针从正北 0° 起算），决定了大气路径的横向取向。

| 值    | 方向 |
| ----- | ---- |
| 0°   | 正北 |
| 90°  | 正东 |
| 180° | 正南 |
| 270° | 正西 |

**对大气模型的影响**：MODTRAN 沿该视线建立三维大气柱坐标系。方位角主要在以下情况有意义：

- 云/气溶胶存在各向异性（如单侧沙尘或烟雾）时，不同方位角的大气透过率略有差异
- 与太阳方位角共同决定"太阳-观测者-目标"之间的相对方位关系，影响太阳杂散光（solar stray light）计算

**推荐取值**：绝大多数辐射制冷场景（天空各向同性）→ **0° 即可**。如材料在特定方向有反射问题（如建筑物遮挡），对应设置。

---

##### 太阳天顶角 (Solar Zenith Angle)

**含义**：太阳相对于观测者头顶的角度。0° = 太阳正上方，90° = 太阳恰好在地平线上。

```
  0° ──────── ☀️（头顶 / 天顶）
    \
    45°        （太阳在45°高度角方向）
      \
      90° ─── 地平线（太阳刚落山）
```

**对大气模型的影响** — 这是最关键的几何参数：

1. **大气路径长度（Air Mass）**：天顶角越大，太阳光穿过的大气层越厚，吸收/散射越强。精确关系为 Air Mass ≈ 1 / cos(天顶角)：

   - 天顶角 0°  → Air Mass = 1（最短路径）
   - 天顶角 60° → Air Mass = 2（2倍路径）
   - 天顶角 80° → Air Mass ≈ 5.76（接近地平线，路径急剧增长）
2. **水汽吸收量**随之变化，直接影响 8-13 μm 窗口的清晰程度
3. 红外热辐射的天空背景受太阳加热大气的影响

**推荐取值**：

| 场景          | 推荐值                  | 说明                                                     |
| ------------- | ----------------------- | -------------------------------------------------------- |
| 正午晴天      | 0°–30°               | 大气路径最短，窗口最干净，对制冷最有利                   |
| 标准白天参考  | 45°                    | MODTRAN 默认值，中午前后的典型状态                       |
| 下午/接近日落 | 60°–75°              | 大气路径变长，窗口变差                                   |
| 早晨刚升起    | 80°–90°              | 大气路径极长，红外窗口被大气遮挡最严重                   |
| 夜间模拟      | >90°（太阳在地平线下） | MODTRAN 自动关闭太阳直射辐射项，但仍存在大气散射天空辐射 |

> ⚠️ **天顶角 > 90° 时**，MODTRAN 会自动禁用太阳直射辐射项，但这不代表大气完全不受太阳影响——大气散射的天空辐射（diffuse sky radiance）依然存在，仍会影响天空亮温度的计算。

---

##### 太阳方位角 (Solar Azimuth)

**含义**：太阳的罗盘方向（顺时针从正北 0° 起算），与观测方位角共同决定太阳在天空中的具体方位。

**对大气模型的影响**：

- 与观测方位角共同决定太阳-观测者-目标之间的相对方位关系
- 影响大气散射光（Rayleigh 散射、Mie 散射）的角度分布计算
- 决定大气路径的昼夜变化时太阳的具体位置

**推荐取值**：

- 仅需计算"大气本身透过率"而不关心具体几何关系 → **0° 即可**
- 如需模拟特定地点、特定时间的大气状态 → 根据日期和经纬度计算
  - 粗略估计：夏至日正午在北纬 40°（如北京）太阳方位角约为 **180°（正南）**
  - 精确值建议使用 Solar Position Algorithm (SPA) 查表或计算工具

---

##### 地面高度 (Ground Altitude)

**含义**：观测点所在的海拔高度（单位：公里），决定了大气的基准气压和气柱总量。

**对大气模型的影响**：

1. **气压与大气密度**：海拔越高，气压越低，大气越稀薄。每升高约 5.5 km，气压约降低一半。
2. **水汽总量减少**：高海拔地区（如青藏高原）水汽含量远低于海平面，大气对 8-13 μm 窗口的吸收更少——这正是高海拔地区辐射制冷效果更好的物理原因。
3. **大气边界层效应**：海平面通常在行星边界层（Planetary Boundary Layer, PBL）内部，气溶胶丰富；高海拔常在边界层之上，大气更清洁。

**典型取值参考**：

| 地点                  | 推荐值      | 效果                                |
| --------------------- | ----------- | ----------------------------------- |
| 海平面城市            | 0–0.5 km   | 大气路径最长，水汽最多，窗口最差    |
| 内陆平原（华北/华东） | 0.5–1.5 km | 较接近默认值 1.5 km                 |
| 高原边缘（兰州/昆明） | 1.5–2.5 km | 水汽明显减少，制冷效果提升          |
| 青藏高原              | 3.0–5.5 km | 最干燥清洁，8-13 μm 窗口透过率最高 |
| 珠峰大本营附近        | >5 km       | 极端干燥，几乎无水汽吸收            |

> 💡 系统前端默认值为 **1.5 km**，这是中国大部分中东部城市的经验折中值（上海/北京海拔约 40 m，但大气柱总量对应高度接近此值）。在高海拔地区研究时，**必须将此值修改为实际海拔**，否则 MODTRAN 会严重高估大气吸收。

---

##### 参数协同作用与典型推荐组合

| 参数                         | 增大时的效果                   | 对 8-13 μm 窗口的影响 |
| ---------------------------- | ------------------------------ | ---------------------- |
| 太阳天顶角 ↑                | 大气路径变长、水汽吸收增加     | 窗口变差               |
| 地面高度 ↑                  | 气压↓、水汽↓、大气更稀薄清洁 | 窗口变好               |
| 气溶胶浓度 ↑                | 散射/吸收增加                  | 全波段变差，窗口也变差 |
| 大气模型（热带→副极地冬季） | 水汽含量递减                   | 窗口变好               |

**典型辐射制冷日间计算的推荐参数组合**：

| 参数       | 推荐值         | 理由                                 |
| ---------- | -------------- | ------------------------------------ |
| 观测方位角 | 0°            | 各向同性天空，无需特别设置           |
| 太阳天顶角 | 45°           | 标准白天参考值                       |
| 太阳方位角 | 0°            | 除非模拟特定地点，否则不影响大气本身 |
| 地面高度   | 与研究地点匹配 | 海平面取 0–0.5 km，高原取实际值     |

### 6.9 功量地图 — 全国各地用这种材料一年能省多少电？

#### 6.9.1 这个问题是什么？

P_net（制冷功率）是瞬时值。但真正有意义的指标是：**一年累计下来，这种材料能省多少电 / 减少多少碳排放？**

**这个工具帮你回答**：「用这种材料在全国各地做屋顶/窗户/衣服，一年能节省多少空调电费？」

#### 6.9.2 能干什么？

基于全球数千个气象站的 **EPW 天气数据**，计算材料在每个城市的**年累计制冷功量和/或年累计制热功量**，生成区域对比地图。

**功量（Wh/m²）是什么？**

- 1 瓦 × 1 小时 = 1 瓦时
- 年功量 = 一年中每个小时的制冷/制热功率的积分
- 通俗理解：**相当于空调用了多少度电（每平方米）**

#### 6.9.3 关键参数：计算方案

| 方案      | 什么时候用                             |
| --------- | -------------------------------------- |
| 纯制冷    | 你只想知道制冷效果（如建筑白天降温）   |
| 纯制热    | 你只想知道制热效果（如太阳能集热器）   |
| 制冷+制热 | 你想知道两者的综合效果（可能相互抵消） |

#### 6.9.4 关键参数：天气组

| 天气组   | 说明         | 典型城市数量 |
| -------- | ------------ | ------------ |
| 中国     | 中国主要城市 | ~30+ 个      |
| 世界     | 全球主要城市 | ~30+ 个      |
| 世界2025 | 最新气象数据 | ~30+ 个      |

#### 6.9.5 关键参数：材料相态配置

这是这个工具最强大的功能：**模拟材料在一年中不同温度下的光学特性变化**。

**场景举例**：你的材料有一种特性：

- 20°C 以下：高发射率（0.95），低太阳吸收（0.3）
- 20-35°C：中等发射率（0.90），中等太阳吸收（0.5）
- 35°C 以上：发射率下降（0.85），太阳吸收增加（0.7）

这种「随温度变化的特性」可以用「相态配置」来描述。

**过渡模式**：

- **渐变（高斯）**：温度变化时，光学参数平滑过渡
- **突变（最近值）**：温度超过某个点就立即切换

#### 6.9.6 典型应用案例

**场景**：你的公司想评估一种新型辐射制冷涂层在全国主要城市的年节能潜力。

**操作**：

1. 选择天气组：china
2. 选择计算方案：cooling（纯制冷）
3. 配置材料相态（填入材料随温度变化的发射率和吸收率）
4. 选择过渡模式：渐变
5. 提交任务（可能需要几分钟到几十分钟）

**输出**：

- 各城市年功量柱状图
- 数据表格（城市、年功量 Wh/m²）
- 可下载 CSV 数据

**解读**：

- 干燥地区（兰州、西宁）：年功量高 → 适合推广
- 潮湿地区（成都、重庆）：年功量低 → 效果有限，可能不适合

- **前端**：`/power-map`
- **后端**：通过 `compare_materials` 任务类型
- **核心脚本**：`material_comparison_tool/examples/compare_materials.py`

这个工具把「个人」和「区域」两个尺度联系起来：

```
单件衣服的制冷功量（W/m²）
        ↓
× 衣服面积（m²/人）
        ↓
= 每个人获得的制冷量（W/人）
        ↓
× 该地区人口数
        ↓
= 该地区的总制冷量（GW）
```

#### 6.11.3 能干什么？

- 基于 EPW 气象数据，计算辐射制冷服饰的年制冷功量
- 结合各省/各城市人口数据，计算区域总节能潜力
- 生成中国省级制冷功量地图

## 七、常见问题与解答

### Q1：计算出来的 P_net 是负数，这是正常的吗？

**A**：完全正常。P_net < 0 意味着净制热——材料在从太阳和大气吸收热量，这正是太阳能热水器所追求的效果。如果你想要制冷效果，就找 P_net > 0 的温度区间。

### Q2：风速云图里，ΔT 是正的还是负的？

**A**：风速云图的 ΔT 定义为「材料温度 - 环境温度」。如果 ΔT < 0，说明材料比环境冷，正在制冷。绝对值越大，制冷效果越好。

### Q3：我的材料发射率很高（0.95），但计算出来的 P_net 却很低，为什么？

**A**：高发射率只是制冷的一个条件。还需要：

- 太阳反射率足够高（白天不会被太阳晒热）
- 大气透过率高（大气窗口干净）
- 风速不能太大
  建议用「功率分量」工具诊断，看是哪个分量在「拖后腿」。

### Q4：应该用哪些大气预设？

**A**：

- 大部分分析：用默认预设就足够
- 高海拔地区：用 MODTRAN 工具自定义
- 极端气候：查阅气象资料，选择最接近的预设
- 有实测大气数据：上传你自己的透过率文件

### Q5：蒸发冷却需要什么条件？

**A**：

- 材料表面必须是亲水的（能让水附着）
- 空气要相对干燥（湿度低，水才能蒸发）
- 空气要流动（风会加速蒸发）
- 如果是户外晴天下使用，还要考虑水膜被太阳晒干的问题

### Q6：为什么有些任务需要很长时间？

**A**：

- 功量地图：需要处理3000+个城市的EPW文件，每个城市8760小时数据，计算量大
- 高精度模式：角度积分步数越多，计算越精确，但越慢
- 原位模拟：从网络下载气象数据的时间取决于网络状况

### Q7：如何引用这个系统？

**A**：如果你在研究中使用了本系统进行计算，建议引用你的计算结果并描述方法。可以参考各模块文档中提供的「论文写法示例」。

### Q8：系统支持透明材料吗？

**A**：支持。如果你的材料是透明/半透明的（如薄膜），可以同时上传：

- 反射率文件：R(λ)
- 透射率文件：T(λ)

系统会自动计算吸收率：α(λ) = 1 - R(λ) - T(λ)

### Q9：启用蒸发潜热后，为什么不同 hc（导热系数）的曲线不再交于环境温度 T_a 这点了？

**A**：这是一个很好的物理观察。回答这个问题需要理解三个层次。

**第一层：为什么不加潜热时，所有 hc 线交于 T_a？**

在不考虑蒸发潜热的情况下，制冷功率公式简化为：

```
P_net = p_r - p_a - h_total × (T_a - T_film) - Q_solar
```

其中 `h_total = h_nat + hc`。当 `T_film = T_a`（即横轴与环境温度相等）时，对流项 `h_total × (T_a - T_film) = 0`，与 `h_total` 的具体取值无关。此时每条曲线在 `T_film = T_a` 处减去相同的值（由辐射项和太阳项决定），因此必然交于一点。

**第二层：加了潜热后，为什么交点就"散开"了？**

蒸发潜热 `Q_latent` 通过 **Lewis 类比**与对流系数挂钩：

```
hc → h_total (= h_nat + hc) → h_m (质量传递系数) → m_dot (质量通量) → Q_latent
```

完整链条如下：

```
hc ──────────────────────────────────────────┐
  ↓                                          │
h_total = h_nat + hc                          │
  ↓                                          │
h_m = h_total / (ρ_air × cp_air × Le^(2/3))  │ Lewis类比
  ↓                                          │
m_dot = h_m × (ρ_v,sat - ρ_v,amb)            │
  ↓                                          │
Q_latent = m_dot × L_v(T_s)                   │
```

也就是说，`Q_latent` 不是一个常数——它**同时依赖于**：
- 表面温度 T_s（通过 `ρ_v,sat` 和 `L_v`）
- 对流条件（通过 `h_total`）

而 `h_nat`（自然对流系数）本身**也是温度的函数**，随温差 `|T_film - T_a|` 增大而增大。因此，不同的 `hc` 值会导致不同的 `h_total`，进而产生不同的 `Q_latent`，使得零点（`P_net = 0` 的点，即湿球温度）不再统一在 `T_a`。

**物理含义**：这个"散开的交点"实际上对应的是**湿球温度**——在湿球温度下，蒸发的潜热损失恰好抵消了净辐射得热。不同的对流条件对应不同的湿球温度。

**第三层：hc=0 的线为什么特别不同？**

如果配置中启用了「自然对流」，则：

| hc 值 | h_total | 含义 |
|-------|---------|------|
| hc=0 | h_nat | 仅有自然对流 |
| hc>0 | h_nat + hc | 自然对流 + 附加导热/强制对流 |

hc=0 代表**纯自然对流**场景，而 hc>0 代表**混合对流**场景。这两种场景的"性格"不同——hc=0 线完全由自然对流的浮力效应主导，而 hc>0 线则叠加了额外的对流增强，因此 hc=0 曲线与其他曲线差异最大是正常的。

> 💡 **如果你希望即使加了潜热，所有 hc 线依然交于 T_a**，可以在任务设置中**关闭「自然对流」选项**。此时 `h_nat = 0`，`h_total = hc` 为常数，蒸发潜热对所有 hc 线的影响相同，交点就会回到 `T_a`。

### Q10：hc（导热系数）的物理含义是什么？为什么需要设置多个值？

**A**：`hc`（导热系数，单位：W/(m²·K)）代表材料与背板/支撑结构之间的热传导附加对流效应，用于模拟材料在实际应用中的热连接状态。

| hc 值 | 典型应用场景 | 说明 |
|-------|------------|------|
| hc=0 | 完全隔热（悬空样品） | 仅靠辐射和自然对流散热 |
| hc=3–6 | 轻质隔热背板 | 材料下方有隔热层，热量难以传递 |
| hc=6–12 | 直接安装在基材上 | 典型屋顶/墙体应用 |
| hc>12 | 金属背板+强通风 | 热量迅速被带走，制冷效果下降 |

设置多个 hc 值是为了模拟材料在不同安装条件下的性能，帮助设计者找到最佳的热管理策略。

P_{net} = P_r - P_a - Q_{conv} - Q_{solar} + P_{phase} + Q_{latent}
$$

| 符号           | 含义               |
| -------------- | ------------------ |
| $P_r$        | 材料向外辐射散热   |
| $P_a$        | 大气向材料辐射加热 |
| $Q_{conv}$   | 对流散热           |
| $Q_{solar}$  | 太阳吸收加热       |
| $P_{phase}$  | 相变潜热（可选）   |
| $Q_{latent}$ | 蒸发潜热（可选）   |

### A.2 斯特藩-玻尔兹曼辐射

$$
P_{rad} = \varepsilon \sigma T^4
$$

| 符号            | 含义                                            |
| --------------- | ----------------------------------------------- |
| $\varepsilon$ | 发射率                                          |
| $\sigma$      | 斯特藩-玻尔兹曼常数 = 5.67×10⁻⁸ W/(m²·K⁴) |
| $T$           | 温度（K）                                       |

### A.3 太阳吸收功率

$$
Q_{solar} = \alpha_s \cdot S_{solar}
$$

| 符号                       | 含义                 |
| -------------------------- | -------------------- |
| $\alpha_s = 1 - R_{sol}$ | 太阳吸收率           |
| $R_{sol}$                | AM1.5 加权太阳反射率 |
| $S_{solar}$              | 太阳辐照度（W/m²）  |

### A.4 大气发射率（经验式）

$$
\varepsilon_{atm} = 1.24\left(\frac{e_a}{T_K}\right)^{1/7} \times corr
$$

其中 $e_a$ 是实际水汽压（kPa），$corr$ 是云量修正系数。

### A.5 对流换热系数（Churchill-Usagi）

$$
h_{conv} = \left(h_{nat}^3 + h_{forced}^3\right)^{1/3}
$$

其中 $h_{nat}$ 和 $h_{forced}$ 分别由瑞利数和雷诺数决定。

### A.6 蒸发潜热（完整计算方案）

当材料表面存在水膜（如亲水涂层、含纤维多孔材料等）时，蒸发过程会带走大量热量，形成额外的制冷效果。本节详细介绍系统中蒸发潜热功率 $Q_{latent}$ 的完整计算流程。

#### A.6.1 整体思路

蒸发潜热功率的计算遵循以下逻辑链：

```
环境温度 T_a（°C）、表面温度 T_s（°C）、相对湿度 RH
         │
         ▼
① 用 Tetens 公式算出饱和水汽压 P_sat(T)
         │
         ▼
② 用理想气体方程把饱和水汽压转为饱和水汽密度 ρ_{v,sat}
         │
         ▼
③ 用 Lewis 类比关系，从对流换热系数 h_conv 推算质量传递系数 h_m
         │
         ▼
④ 质量通量 ṁ = h_m × (ρ_{v,sat} − ρ_{v,amb})
         │
         ▼
⑤ 乘以温度相关的汽化潜热 L_v(T) 得到 Q_{latent}
         │
         ▼
⑥ 乘以湿润面积比例 wet_fraction，得到有效蒸发潜热功率
```

#### A.6.2 第一步：Tetens 公式计算饱和水汽压

饱和水汽压是指空气在给定温度下所能容纳的最大水蒸气分压力（单位：kPa），采用 Tetens 经验公式：

$$
P_{sat}(T) = 0.61078 \exp\!\left(\frac{17.27\, T}{T + 237.3}\right) \quad \text{[kPa]}
$$

其中温度 $T$ 的单位是摄氏度（°C）。

**代入两个温度**，分别得到表面和环境的饱和水汽压：

$$
P_{sat,s} = P_{sat}(T_s - 273.15), \quad P_{sat,a} = P_{sat}(T_a - 273.15)
$$

**环境实际水汽压**由相对湿度 RH（0–1 之间的小数）决定：

$$
P_v = RH \times P_{sat,a}
$$

#### A.6.3 第二步：水汽压转换为水汽密度

将 kPa 换算为 Pa，然后使用理想气体状态方程，把水汽压转换为水汽密度（单位：kg/m³）：

$$
\rho_{v,sat} = \frac{P_{sat,s} \times 1000}{R_v \, T_s}, \quad
\rho_{v,amb} = \frac{P_v \times 1000}{R_v \, T_a}
$$

其中 $R_v = 461.5\ \mathrm{J/(kg\times K)}$ 是水蒸气的气体常数。


**直观理解**：密度差 $(\rho_{v,sat} - \rho_{v,amb})$ 代表"表面能容纳的水汽量"与"环境实际水汽量"之差。如果表面附近空气比环境更干燥，水就会从表面蒸发，带走热量。

#### A.6.4 第三步：Lewis 类比求质量传递系数

对流换热系数 $h_{conv}$（单位：W/(m²·K)）描述热量传递效率，而质量传递系数 $h_m$（单位：m/s）描述水汽传递效率。两者通过 **Lewis 类比**联系起来：

$$
h_m = \frac{h_{conv}}{\rho_{air} \cdot c_{p,air} \cdot Le^{2/3}}
$$

各项含义如下：

| 符号 | 含义 | 典型值 |
|------|------|--------|
| $h_{conv}$ | 对流换热系数 | 由系统其他模块计算（如 5.0 W/(m²·K)） |
| $\rho_{air}$ | 空气密度（kg/m³） | $1.225 \times \frac{273.15}{T_{film}}$（理想气体近似，$T_{film}$ 为膜温度 K） |
| $c_{p,air}$ | 空气比热容 | $1005\ \mathrm{J/(kg\times K)}$ |
| $Le$ | Lewis 数 | $0.84$（水蒸气–空气系统实测值） |

膜温度取表面与环境温度的平均值：$T_{film} = (T_s + T_a) / 2$。

> **为什么要用 Lewis 类比？** 因为热量传递和水汽传递共享同一种输运机制（分子扩散+湍流扩散），Lewis 数把两者联系起来，使得只要知道热传递效率 $h_{conv}$，就能推算出水汽传递效率 $h_m$。这是一个在暖通空调和蒸发冷却领域广泛使用的经验关系。

#### A.6.5 第四步：计算质量通量

质量通量（单位：kg/(m²·s）表示单位面积、单位时间内蒸发的水质量：

$$
\dot{m} = h_m \, \bigl(\rho_{v,sat} - \rho_{v,amb}\bigr)
$$

- 当 $\rho_{v,sat} > \rho_{v,amb}$（表面比环境干燥）：$\dot{m} > 0$，水蒸发 → 带走热量，$Q_{latent} > 0$（制冷）
- 当 $\rho_{v,sat} < \rho_{v,amb}$（表面比环境潮湿）：$\dot{m} < 0$，水凝结 → 释放热量，$Q_{latent} < 0$（制热）

#### A.6.6 第五步：汽化潜热与蒸发功率

水的汽化潜热 $L_v$ 是指单位质量的液态水变成水蒸气所需的热量（单位：J/kg）。它随温度略有变化，代码中使用以下经验公式（0–100°C 范围）：

$$
L_v(T) = \bigl(2501.0 - 2.36\, T - 0.0016\, T^2\bigr) \times 1000 \quad \text{[J/kg]}
$$

其中 $T$ 是摄氏温度（°C）。

温度相关的汽化潜热参考值：

| 温度（°C） | $L_v$（MJ/kg） |
|-----------|----------------|
| 0         | 2.501          |
| 20        | 2.446          |
| 40        | 2.407          |
| 60        | 2.356          |
| 80        | 2.303          |

最终，**单位面积的蒸发潜热功率**（单位：W/m²）为：

$$
Q_{latent} = \dot{m} \times L_v(T_s)
$$

#### A.6.7 第六步：湿润面积比例

实际材料表面通常只有一部分被水膜覆盖（例如纤维之间的孔隙、亲水区域等）。引入湿润面积比例 $\text{wet\_fraction} \in [0, 1]$：

$$
Q_{latent,eff} = \text{wet\_fraction} \times Q_{latent}
$$

- $\text{wet\_fraction} = 1$：整个表面被水膜覆盖（理想情况）
- $\text{wet\_fraction} = 0.5$：只有一半面积有水膜蒸发
- 亲水性强的材料、含多孔纤维的材料通常 $\text{wet\_fraction}$ 较大

#### A.6.8 完整公式汇总

综合以上六个步骤，蒸发潜热功率的完整表达式为：

$$
\boxed{
Q_{latent,eff} = \text{wet\_fraction} \times h_m \bigl(\rho_{v,sat} - \rho_{v,amb}\bigr) \times L_v(T_s)
}
$$

其中：

$$
\begin{aligned}
P_{sat}(T) &= 0.61078 \exp\!\left(\frac{17.27\, T}{T + 237.3}\right) & \text{[kPa]} \\[4pt]
\rho_{v,sat} &= \frac{P_{sat,s} \times 1000}{R_v \, T_s}, \quad \rho_{v,amb} = \frac{RH \times P_{sat,a} \times 1000}{R_v \, T_a} & \text{[kg/m³]} \\[4pt]
h_m &= \frac{h_{conv}}{\rho_{air} \cdot c_{p,air} \cdot Le^{2/3}} & \text{[m/s]} \\[4pt]
L_v(T) &= \bigl(2501.0 - 2.36\, T - 0.0016\, T^2\bigr) \times 1000 & \text{[J/kg]}
\end{aligned}
$$

#### A.6.9 典型数值范围参考

| 参数 | 典型范围 | 说明 |
|------|---------|------|
| $P_{sat}$ | 0.6–100 kPa | 0°C 到 100°C 饱和水汽压 |
| $\rho_{v,sat} - \rho_{v,amb}$ | 0–0.02 kg/m³ | 水汽密度差，干燥时大 |
| $h_m$ | 0.001–0.02 m/s | 自然对流→强制通风 |
| $L_v$ | 2.3–2.5 MJ/kg | 随温度降低 |
| $Q_{latent,eff}$ | 0–200 W/m² | 蒸发制冷功率上限 |

💡 **简化估算**：在夏季典型条件下（30°C、相对湿度 50%、有轻微通风），$Q_{latent,eff}$ 约在 30–80 W/m² 之间，相当于一块高效辐射制冷材料纯辐射项的 30%–50%，是相当可观的附加制冷贡献。

#### A.6.10 适用条件与局限性

| 适用场景 | 不适用场景 |
|---------|-----------|
| 亲水/超亲水材料表面 | 疏水材料（无水膜） |
| 相对湿度 RH < 80% | 高湿度（RH > 90%）时蒸发变慢 |
| 有风或强制通风 | 完全无风且高湿的封闭环境 |
| 温度 > 0°C | 结冰温度以下（冰升华量极小） |

### 附录B 材料设计指南 — 理想光谱特征与常见误区

#### B.1 理想日间辐射制冷材料的光谱特征

一个优秀的日间辐射制冷材料，其光谱曲线应具备以下两个特征：

**太阳波段（300-2500 nm）——高反射率**

太阳反射率 R_sol 应尽可能接近 1.0，理想目标 > 0.93。可行技术路线：

| 技术路线                          | R_sol 范围 | 优点             | 缺点                 |
| --------------------------------- | ---------- | ---------------- | -------------------- |
| 金属介质复合镜面（Al/Ag + SiO₂） | 0.93-0.97  | 反射率最高       | 成本高、脆性大       |
| 白色颜料涂层（TiO₂、ZnO）        | 0.85-0.92  | 成本低、工艺成熟 | 发射率也较高         |
| 光子晶体/超材料                   | 0.96-0.99  | 反射率极高       | 实验室阶段、成本极高 |
| 微纳结构表面                      | 0.88-0.94  | 可大面积制备     | 长期稳定性待验证     |

**红外窗口波段（8-13 μm）——高发射率**

在 8-13 μm 窗口内发射率应接近 1.0。可行材料：SiO₂（~0.90）、ZrO₂（~0.87）、Al₂O₃（~0.83）、聚合物薄膜 PE/PMP（~0.87-0.93）。

**专家级设计——选择性发射**

在 8-13 μm 窗口内高发射率的同时，在非窗口红外波段（< 8 μm 和 > 13 μm）适当抑制发射率，可减少"大气反加热"（大气非窗口红外辐射被材料吸收）。理论上可将日间净制冷功率提升 15-30 W/m²，但工程实现较复杂。

#### B.2 多层结构设计原则

实际应用的辐射制冷材料几乎都采用多层结构，三种典型模式：

**模式A：太阳反射层 + 红外发射层**

```
[顶层] 高太阳反射层（金属基底 + 介质抗反射层）
[底层] 高红外发射材料（SiO₂ 涂层等）
```

适用于：建筑屋顶、冷水管道、室外设备表面。

**模式B：双功能单层**
在单一材料层内同时实现高太阳反射和高红外发射。如白色 TiO₂ 颜料涂层（R_sol ≈ 0.85，ε ≈ 0.90）。适用于：薄膜应用、成本敏感场景。

**模式C：透明型（用于玻璃/光伏板）**
涂层在可见光（380-780 nm）必须高度透明（透射率 > 0.90），同时在 8-13 μm 具有高发射率（> 0.80）。日间制冷功率通常有限（~10-30 W/m²），但对光伏板有额外降温效果。

#### B.3 六大常见设计误区

> **误区一：只追求高发射率，忽视太阳反射率**

这是最常见的错误。很多材料（如黑漆）红外发射率接近 1.0，但在太阳波段的吸收率也接近 1.0，导致白天无法制冷。正确优先级：**太阳反射率 > 红外发射率 > 大气窗口选择性**。

> **误区二：忽视对流热损失**

初学者往往只关注辐射项而忽视对流。在有风环境下，对流热损失可以完全抵消辐射制冷效果。设计时必须考虑安装位置的风环境，必要时增加防风罩或隔热支架。

> **误区三：认为发射率 ε = 1.0 是最优的**

当发射率 ε = 1.0 时，非窗口波段的"大气反加热"也随 ε 线性增加。在潮湿地区，制造 ε = 0.90 的材料成本远低于 ε = 1.00，但实际效果差距可能不超过 5%。

> **误区四：用夜间性能推断白天性能**

夜间（无太阳辐射）测试表现良好的材料，白天可能完全失效。必须分别评估昼/夜性能，不能用夜间数据推断白天。

> **误区五：认为 AM1.5 加权反射率不重要**

不同地区的太阳光谱差异可达 10-15%。在多尘或高海拔地区，实际太阳光谱偏向紫外和蓝光方向，若材料的反射峰恰好落在这些区域，R_sol 可能显著偏离预期值。本系统使用标准 AM1.5 光谱，如需精确匹配特定地区，应使用当地实测太阳光谱。

> **误区六：高估相变材料（PCM）的制冷效果**

PCM 在特定温度下吸收/释放潜热，可以作为温度调节的辅助手段，但 PCM 也带来额外的热阻层，且仅在材料温度接近相变点时才工作。在辐射制冷设计中，PCM 通常是"锦上添花"而非"雪中送炭"。
